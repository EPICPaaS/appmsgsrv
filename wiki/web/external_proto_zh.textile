<h3>Terry-Mao/gopush-cluster Web http协议文档</h3>
Web外部接口文档，用于获取Subcribe节点,获取离线消息等

<h3>接口汇总</h3>
(head). | 接口名 | URL | 访问方式 |
| "获取订阅节点":ServerGet | /server/get | GET |
| "获取离线消息":MsgGet    | /msg/get    | GET |

<h3>公共返回码</h3>

(head). | 错误码 | 描述 |
| 0 | 成功 |
| 65534 | 参数错误 |
| 65535 | 内部错误 |

<h3>基本返回结构</h3>
<pre>
{
    "ret": 0,  //错误码
    "data": {….}//如果有数据返回，则有data，无数据返回就只有前面的ret和msg
}
</pre>
如何接口需要返回jsonp结构(例如客户端使用js解析json)，则指定callback参数，例如：callback=test，则返回结构如下：
<pre>
test({
    "ret": 0,  //错误码
    "data": {….}//如果有数据返回，则有data，无数据返回就只有前面的ret和msg
})
</pre>

<h3>获取订阅节点</h3>
 * 请求参数

(head). | 参数 | 类型 | 描述 |
| key   | string | 订阅key |
| proto | int    | 订阅协议 1:websocket 2:tcp |
| callback   | string | 返回jsonp结构函数名(可选) |

 * 返回参数说明

(head). | 参数 | 类型 | 描述 |
| server | string | 返回的可用于订阅的地址 |

 * 返回码

(head). | 错误码 | 描述 |
| 1001 | 没有找到key对应的节点 |
| 1003 | 未知的订阅协议 |
<pre>
{
    "ret": 0,
    "data": {
	    "server": "127.0.0.1:8080"  //IP:Port
    }
}
</pre>

<h3>获取离线消息</h3>
 * 请求参数

(head). | 参数 | 类型 | 描述 |
| key  | string | 订阅key |
| mid  | int64  | 最新接收的私有消息ID |
| pmid | int64  | 最新接收收的公共消息ID |
| callback   | string | jsonp函数名(可选) |

 * 返回参数说明

(head). | 参数 | 类型 | 描述 |
| msgs  | string数组 | 私有离线消息 |
| pmsgs | string数组 | 公共离线消息 |
注：
1.解析每条message的时候注意，每条信息是string数据，而不是结构体
2.返回msgs、pmsgs消息中的参数mid类型为int64.

 * 返回结果

<pre>
{
    "data": {
        "msgs": [
            {"msg":"{\"test\":1}","mid":13999084541846408,"gid":0},
            {"msg":"{\"test\":2}","mid":13999084579056605,"gid":0}
        ],
        "pmsgs": []
    },
    "ret": 0
}
</pre>
